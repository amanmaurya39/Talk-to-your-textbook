# ADR-002: Google Gemini as Generation Model

**Date:** July 2026  
**Status:** Accepted  

---

## Context

The Talk To Your Textbook app needs a Language Model (LLM) to:
- Generate grounded, accurate answers to questions based on retrieved textbook chunks
- Produce reliable page-level citations for every statement made
- Handle a large context window to process retrieved page passages
- Run efficiently on a development environment with no cost and minimal setup overhead

I evaluated GPT-4o-mini, Google Gemini (via AI Studio free tier), and local open-source models (via Ollama).

---

## Decision

**Use Google Gemini (specifically `gemini-2.5-flash`).**

---

## Reasons

- **Free Tier Availability:** Google AI Studio offers a generous free tier (up to 15 RPM and 1 million TPB) with no credit card required, unlike OpenAI's models which require prepaid credits.
- **Strong Instruction-Following:** Gemini Flash models demonstrate high-quality instruction following, which is critical for enforcing the system prompt rules (e.g. strict page citations like `[Page X]` and refusing out-of-context questions).
- **Fast Execution:** Gemini 2.5 Flash is highly optimized for low latency, giving users near-instant responses in the Streamlit UI.
- **Large Context Window:** Supports up to 1 million tokens, easily accommodating multiple retrieved pages of textbook text.

---

## Consequences

**Positive:**
- Zero ongoing cost for students and developers during development and testing
- Simple setup: requires only a Google AI Studio API key (configured in `.env`)
- Reliable citation formatting matching the requested system prompt guidelines
- No heavy GPU or hardware required on the host system

**Negative:**
- Internet connection required to access the Google AI Studio API
- Subject to free tier rate limits (max 15 requests/minute), which can cause temporary rate limit errors during intensive evaluation runs
- Uploaded contexts are sent to Google (though standard AI Studio data terms apply)

---

## Alternatives Considered

| Option | Why Rejected |
|---|---|
| **GPT-4o-mini** | Requires a paid OpenAI account with prepaid API credits, making it harder for students to run for free. |
| **Local Llama 3 (via Ollama)** | Requires substantial local CPU/GPU resources and memory, which may run very slowly or fail to launch on standard student laptops. |
| **Hugging Face Serverless API** | Unreliable rate limits, higher latency, and less stable for production-like student applications. |

---

## Review (3rd Year)

If this project scales up to a commercial or multi-user SaaS product, migrate to a paid production tier of Google Gemini or OpenAI APIs to resolve rate limits, or deploy a dedicated fine-tuned open-source model on cloud infrastructure.
